## Eliminated invalid states (for the ledger tick at merge)

- "-1 means none" is unrepresentable in the core: visibleIndexOf /
  clientVisibleIndexOf / first-lastVisibleIndex / assignedLayoutIndex
  return std::optional<int>, hiddenSkippingNeighbor returns
  std::optional<RowEntry>; the -1 convention survives only at the QML
  boundary in the wrapper (the EX-01/EX-02 pattern).
- A hidden or missing entry claiming visible indexes through -1 base
  arithmetic is unrepresentable: belongsAtVisibleIndex composes over
  the optional base, so Qt5's [-1, count-1) range bug (shipped QML
  recorded answering true for a hidden client at v0/v1; an unindexed
  ParabolicEdgeSpacer claiming v1) cannot be expressed.
- Walk direction is enum class Direction (Tail/Head), not the raw
  +/-1 step ints the four QML twins hand-rolled; the wrapper validates
  the QML-side int and refuses anything else loudly.
- The collector contract (index >= 0, subItemCount >= 0) is asserted
  at every core entry point (live under QT_FORCE_ASSERTS) and refused
  loudly at the wrapper boundary (router model); a default RowEntry is
  the documented not-yet-indexed state and trips the assert if it ever
  reaches a row.
- Four hand-drifted twins are now one implementation: the drift state
  itself is gone (twinCoherence unit slot + the generator's 14-vector
  equality runs across both cutovers pin it).
